# [AWS] Cognito

# 개요

Amazon Cognito는 웹 및 모바일 앱에 대한 인증, 권한 부여 및 사용자 관리를 제공합니다. 기본적인 방식인 아이디와 패스워드를 사용하여 직접 로그인하거나 Facebook, Amazon, Google 또는 Apple 같은 소셜 로그인 기능을 제공하는 서비스를 통해 로그인할 수 있습니다.

Cognito는 사용자 풀(user pool) 과 **자격 증명 풀(identity pool)**로 구성되어 있습니다. 이 둘은 별도로 또는 함께 사용할 수 있습니다.

# 기능

# 사용자 풀 (user pool)

사용자 풀에서 사용자는 Amazon Cognito , 또는 **서드 파티 자격 증명 공급자(idP)**를 통해 웹/모바일 앱에 로그인할 수 있습니다. 어떤 방식으로 로그인하는지 여부와는 무관하게 사용자 풀의 모든 멤버는 각각의 프로필을 보유하게 되며 SDK를 통해 엑세스 할 수 있습니다.

사용자 풀 제공 사항:

  • 가입 및 로그인 서비스
  • 사용자 로그인을 위한 내장 사용자 지정 웹 UI. (그냥 간단한 로그인 화면)
  • Facebook, Google, Login with Amazon 및 Sign in with Apple을 통한 소셜 로그인 및 사용자 풀의 SAML 및 OIDC 자격 증명 공급자를 통한 로그인
  • 사용자 저장소 관리 및 사용자 프로필
  • 멀티 팩터 인증(MFA), 이상 있는 자격 증명 확인, 계정 탈취 보호, 전화 및 이메일 확인과 같은 보안 기능
  • AWS Lambda 트리거를 통한 사용자 지정 워크플로우 및 사용자 마이그레이션

**AWS Lambda**는 AWS에서 제공하는 서버리스 컴퓨팅 서비스입니다. 서버리스 컴퓨팅은 애플리케이션을 실행하기 위한 **별도의 서버 셋업 없이 곧바로 코드를 실행시켜주는 서비스**를 의미하며, 고정 비용 없이 사용 시간에 대해서만 비용이 발생합니다. **AWS EC2**는 현재 초 단위로 계산되는 반면 **AWS Lambda**는 1ms당 요금을 계산해 사용한만큼만 비용이 발생 됩니다.

# 자격 증명 풀(identity pool)

자격 증명 풀로 사용자는 **임시 AWS 자격 증명**을 얻어 S3, DynamoDB 같은 AWS 인프라에 접근권한을 부여 할 수 있습니다.

자격 증명 풀을 지원하는 사용자:

  • Cognito user pool 사용자
  • Facebook, Google, Apple, Amazon 을 통한 소셜 로그인 사용자
  • OpenId Connect(OIDC) , Security Assertion Markup Language(SAML) 자격 증명 공급자
  • 개발자 인증 자격 증명

사용자 프로필 정보를 저장하려면 자격 증명 풀이 사용자 풀에 통합되어야 합니다.
