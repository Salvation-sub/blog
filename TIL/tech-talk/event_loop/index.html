# 비동기 코드 예시

console.log("집에 들어가면 전화를 주겠다고 하셨는데, 연락이 없으셔서요!");

// 자바스크립트는 setTimeout 같은 비동기 코드를 만나면 자바스크립트의 뒷편(backstage)에서 실행한 다음 나머지 코드를 진행합니다.
setTimeout(function () {
  console.log("저기요?..");
}, 10000);

console.log("노숙하시는 건가요?");

// 1. 집에 잘 들어가셨나요?  2. 노숙하시는건가요?   3. 저기요..? 순으로 실행됩니다.

# 콜백 함수

  • 다른 함수의 인자로 넘겨지는 함수
  • 콜백 수신 함수에 의해 특정 시점에 실행
  • 동기 콜백의 경우 호출 즉시 실행
  • 비동기 콜백의 경우 나중에 조건을 만족했을 때 실행 (setTimeOut)
  • 이벤트 리스너
  • 타이머 / XMLHttpRequest 요청

# 자바스크립트 엔진

  • Webkit , V8
  • 자바스크립트 코드를 해석하고 실행하는 인터프리터
  • 힙(Heap) + 호출스택(call Stack)
  • 힙: 메모리 할당이 일어나는 부분(변수, 객체 등이 저장되는 일종의 창고)
  • 호출스택(실행 컨텍스트) :
    • 함수가 호출되는 순서대로 쌓이는 스택
    • 함수 실행 시 호출 스택에 해당 함수를 집어넣음
    • 함수 return시 (함수의 실행이 끝나면) 호출 스택의 맨 위에 있는 해당 함수를 끄집어냄
  • 자바스크립트는 싱글 스레드 언어 = 호출 스택을 하나만 사용한다.

# 브라우저의 구조

  • 자바스크립트 엔진 : 힙 + 콜스텍
  • Web APIs(멀티쓰레드) : DOM(document) , AJAX(XMLHttpRequest), Timeout(setTImeout)가 브라우저에서 제공됨
  • Callback Queue(Task Queue) : Web API의 콜백 함수들이 대기하는 큐
  • 이벤트 루프(EventLoop) :
    • 호출 스택과 콜백 큐를 계속 주시하고 있습니다.
    • 호출 스택이 비어있으면(꼭 비어 있어야함!), 먼저 들어온 순서대로 콜백 큐에 있는 콜백 함수를 호출 스택으로 집어 넣습니다.
